一、产品定位
================================================================================

一句话：中国最全的资格考试信息+社区平台，让每个人都能找到值得考的证，
        并知道怎么考、值不值得考。

定位关键词：信息工具、内容丰富、社区驱动、AI辅助

区别于培训机构网站的差异点：
- 信息中立客观，不做培训推荐
- 官方通知原文引用+AI摘要，信息可信且高效
- 多维度分类体系，按"我当前阶段能考什么"来导航
- 社区真实评价，用过来人的经验帮新人避坑

三、核心功能模块

────────────────────────────────────────────────────────────────────────────────
模块A：考试信息引擎（核心）
────────────────────────────────────────────────────────────────────────────────

每个考试的信息卡片包含：

[基础档案]
├─ 考试全称 / 简称
├─ 主办单位 / 发证机构
├─ 官方网站
├─ 证书类别标签（准入类/水平评价类/国际认证/企业认证/技能等级）
├─ 行业标签（财会/法律/建筑/医疗/教育/IT/...）
├─ 含金量评级（1-5星，社区评分驱动）
├─ 难度评级（1-5星）
├─ 一句话简介

[报名信息]
├─ 报名条件摘要（学历/专业/工作经验要求）
├─ 报名时间节点（通常每年几月）
├─ 报名入口（官网链接）
├─ 报名费用

[考试信息]
├─ 考试科目列表
├─ 考试形式（笔试/机考/面试/实操）
├─ 考试时间节点（通常每年几月）
├─ 成绩公布时间
├─ 证书有效期

[数据洞察]
├─ 近年报考人数趋势
├─ 近年通过率
├─ 持证人数
├─ 薪资影响（如有数据来源）

[官方通知时间线]
├─ 按时间倒序展示所有官方公告
├─ 每条：标题 + AI摘要 + 原文链接 + 发布时间
├─ 关键信息自动提取高亮（报名时间/条件变更/大纲调整）

维度1：管理属性

├─ 企业认证
├─ 技能等级认定
└─ 其他社会认可证书

维度2：行业领域
├─ 财会金融 / 法律 / 建筑地产 / 医疗健康
├─ 教育培训 / IT互联网 / 机械制造 / 能源环保
├─ 交通运输 / 农林牧渔 / 文化艺术 / 公共管理
└─ 可扩展

维度3：适用人群
├─ 在校生可考 / 应届推荐
├─ 1-3年经验 / 3-5年经验 / 5年以上
├─ 不限专业 / 限相关专业
└─ 转行入门 / 晋升加分 / 政策刚需

├─ 多维筛选器（行业×类型×人群×时间）

├─ 考试日历（时间线/月历视图）
├─ 考试地图（按主管部门/按地域分布）

五、UI设计方向：内容丰富型
================================================================================

参考风格：小红书（图文社区感）+ 知乎（深度内容感）

设计原则：
├─ 卡片化：每种考试/通知/帖子用卡片承载，视觉层次分明
├─ 大图引导：考试卡片配封面图（官方Logo或行业代表图）
├─ 标签系统：彩色标签快速传达分类信息
├─ 数据可视化：通过率/热度用简单图表替代纯数字
├─ 间距呼吸感：留白充足，一屏信息量适中，避免拥挤
├─ 微交互：卡片点击缩放、下拉回弹、Tab切换过渡动画
└─ 暗黑模式支持

配色方向：
├─ 主色：深蓝（专业/信任感）+ 暖色点缀（橙/金色用于星级和标签）
├─ 背景：浅灰（卡片在白色背景上浮起）
└─ 文字：深灰而非纯黑（阅读舒适）

字体：系统默认中文字体（PingFang SC / Noto Sans SC）

鉴于"内容丰富型"UI需求 + 单人开发 + "都可以"的技术偏好，
推荐以下组合（平衡开发效率和长期扩展性）：

┌──────────────────────────────────────────────────────┐
│  层级        │  技术              │  理由              │

│             │                    │  好，Feed流性能优  │
├──────────────────────────────────────────────────────┤
│  后端API    │  Python FastAPI    │  爬虫生态 + AI集成 │
│             │                    │  顺手               │
├──────────────────────────────────────────────────────┤
│  数据库      │  PostgreSQL 16     │  结构化+JSONB标签  │
│             │                    │  +全文搜索          │
├──────────────────────────────────────────────────────┤
│  缓存        │  Redis             │  热门数据缓存       │
├──────────────────────────────────────────────────────┤
│  爬虫        │  Scrapy +          │  成熟稳定的爬虫     │
│             │  Playwright        │  框架 + 动态页面    │
├──────────────────────────────────────────────────────┤
│  AI服务      │  Claude API        │  公告解析/摘要     │
│             │                    │  /标签建议          │

替代方案：
├─ 如果Flutter学习成本高 → React Native + Node.js (NestJS)
└─ 如果不想管服务器 → Supabase(数据库+后端) + Vercel(部署)

七、核心数据模型
================================================================================

-- 考试主表
exam (
  id              UUID PRIMARY KEY,
  name            VARCHAR(100) NOT NULL,       -- 简称
  full_name       VARCHAR(200),                -- 全称
  category        VARCHAR(50),                 -- 准入类/水平评价类/国际认证/企业认证/技能等级
  industry        VARCHAR(50),                 -- 财会/法律/建筑/医疗/教育/IT/...
  target_group    JSONB,                       -- ["在校生","应届","职场1-3年"]
  organizing_body VARCHAR(200),                -- 主办单位
  official_website VARCHAR(500),               -- 官网URL
  difficulty      INT CHECK(1-5),             -- 难度评级
  gold_content    DECIMAL(2,1),               -- 含金量评分(社区驱动)
  description     TEXT,                        -- 简介
  eligibility     TEXT,                        -- 报名条件摘要
  subjects        JSONB,                       -- [{name, format, duration}]
  tags            JSONB,                       -- 扩展标签
  logo_url        VARCHAR(500),
  status          VARCHAR(20) DEFAULT 'active',-- active/deprecated
  created_at      TIMESTAMP,
  updated_at      TIMESTAMP
)

-- 时间节点表
exam_timeline (
  id              UUID PRIMARY KEY,
  exam_id         UUID REFERENCES exam(id),
  event_type      VARCHAR(30),                -- 报名开始/报名截止/准考证打印/考试/出成绩
  period_desc     VARCHAR(200),               -- "每年6月" 或 "上半年/下半年"
  date            DATE,                        -- 具体日期(如已知)
  is_estimated    BOOLEAN DEFAULT true,       -- 是否预估
  source_url      VARCHAR(500)
)

-- 官方通知表
exam_notice (
  id              UUID PRIMARY KEY,
  exam_id         UUID REFERENCES exam(id),
  title           VARCHAR(500),
  publish_date    DATE,
  summary         TEXT,                        -- AI摘要
  key_points      JSONB,                       -- [{type, content}] 提取的关键信息
  source_url      VARCHAR(500),
  fetched_at      TIMESTAMP
)

-- 用户表 (Phase 2)
app_user (
  id              UUID PRIMARY KEY,
  phone           VARCHAR(20) UNIQUE,
  nickname        VARCHAR(50),
  avatar_url      VARCHAR(500),
  career_stage    VARCHAR(30),                -- 在校/应届/职场1-3年/职场3-5年/职场5年+
  industry        VARCHAR(50),
  interests       JSONB,                       -- 感兴趣的行业列表
  created_at      TIMESTAMP
)

-- 关注表 (Phase 2)
user_follow (
  user_id         UUID REFERENCES app_user(id),
  exam_id         UUID REFERENCES exam(id),
  remind_signup   BOOLEAN DEFAULT true,       -- 报名提醒
  remind_exam     BOOLEAN DEFAULT true,       -- 考试提醒
  remind_score    BOOLEAN DEFAULT true,       -- 出分提醒
  created_at      TIMESTAMP,
  PRIMARY KEY (user_id, exam_id)
)

八、分阶段实施计划
================================================================================

────────────────────────────────────────────────────────────────────────────────
Phase 1（4-6周）：核心信息工具 MVP
────────────────────────────────────────────────────────────────────────────────

目标：单人可完成，验证"考试信息聚合"这件事有没有人用

范围：
✅ Flutter App（iOS + Android）
✅ 本地SQLite数据库嵌入初始数据（不需要后端服务器）
✅ 30-50个热门考试的手动精选数据
✅ 考试搜索（名称/行业/关键词）
✅ 多维度筛选浏览（行业×类型×人群）
✅ 考试详情页（概览Tab：基本信息+报名条件+考试科目+时间节点）
✅ 考试日历视图（月历+列表切换）
✅ 关注考试（本地存储）
✅ 本地通知提醒（报名/考试/出分）
✅ 收藏功能（本地存储）
✅ 浏览历史

❌ 不做：
  ✗ 后端服务器
  ✗ 用户注册登录
  ✗ 社区功能
  ✗ 爬虫自动化
  ✗ AI信息提取
  ✗ 推送通知
  ✗ 数据可视化图表
  ✗ 个性化推荐

数据来源：手动从官网收集，结构化录入JSON/SQLite文件，打包进App

────────────────────────────────────────────────────────────────────────────────
Phase 2（4-6周）：后端+自动化
────────────────────────────────────────────────────────────────────────────────

目标：数据可持续更新，用户可注册登录

新增：
✅ Python FastAPI 后端
✅ PostgreSQL 数据库
✅ 用户注册/登录（手机号验证码）
✅ 云端同步关注列表+收藏
✅ 后台定时爬虫（10个重点考试官网，每日检查）
✅ AI公告解析（Claude API：公告PDF/网页 → 结构化数据）
✅ 推送通知（关注的考试有新公告/时间节点临近）
✅ 几个基础图表（通过率趋势等）
✅ 管理后台（审核AI提取的信息）

────────────────────────────────────────────────────────────────────────────────
Phase 3（6-8周）：社区+完整生态
────────────────────────────────────────────────────────────────────────────────

新增：
✅ 考试评价系统（打分+评论）
✅ 经验分享（发帖+富文本）
✅ 问答模块
✅ 圈子（行业圈+考试圈）
✅ 备考打卡
✅ 内容质量体系

十、关键设计决策记录
================================================================================

已确认：
  ✅ 平台形态：移动端App（iOS + Android，Flutter跨平台）
